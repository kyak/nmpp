85 // The functions listed below are forward and inversed FFT routines for
86 // 256,512,1024 or 2048-point compex data, represented as arrays of nm32sc type.
87 // Each complex number is stored in 64-bit word.
88 // The lower 32-bits is real part of complex number.
89 // The higher 32-bits is imaginary part of complex number;
90 // The admissible input range of data depends on dimension of array,
91 // mode of calculation accuracy and on/off mode of intermediate and final scaling down (shift normalization) of results.
92 // This range guarantee against overflow during calculation process.
93 // The table of ranges you may find in "FFT Library Programmer's manual"
94 //
95 // The mode of calculation accuracy tells how sine-cosine coeffecients are represented if fixed-point format.
96 // When the 7-bit accuracy mode is used, output shape accuracy approaches of the maximum,
97 // but output is reduced by around 2%.
98 // If the 6-bit accuracy mode is used, then output range corresponds to result of Fourie transform
99 // based on floating-point arithmetic, but output is less precise.
100 // The mode of calulation accuracy may be set or switched by appropriate ***Set6bit() or ***Set7bit() function
101 // NOTE: At least on time the accuracy setting function must be called before FFT routine executing.
